# Code smells — Change preventers

A change in one requirement forces edits in many places, or subclasses must evolve in lockstep. Cost of change is high; fear of regressions slows delivery.

**Heuristic:** “we can’t touch that file—everything breaks” often maps to **shotgun surgery** or **divergent change** (one module changing for many unrelated reasons).
