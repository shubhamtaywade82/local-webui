# SOLID (cross-language)

Short reminders aligned with common teaching (including [Refactoring.Guru](https://refactoring.guru/) materials on principles and refactorings):

- **S — Single Responsibility:** one reason to change per module; split mixed concerns.
- **O — Open/Closed:** extend behavior via new types or composition rather than editing stable core logic everywhere.
- **L — Liskov Substitution:** subtypes must honor contracts of supertypes; no surprising pre/postconditions.
- **I — Interface Segregation:** small, focused interfaces so clients do not depend on unused methods.
- **D — Dependency Inversion:** depend on abstractions; high-level policy should not depend on low-level details.
